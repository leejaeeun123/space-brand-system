> 로고·디스플레이·본문 전부 Paperlogy. 이전 **Pretendard 단일**(2026-06-28) 및 League Spartan 서술은 **폐기**.
> 재은 결정: “Pretendard는 너무 흔하다 → 정돈된(각진·기하)이면서 포인트가 되는 폰트로.” Paperlogy·NanumSquare 2파이널 중 **Paperlogy 확정**.
> **🟢 2026-07-06 굵기 통일**: "로고가 두껍다" 피드백 → **볼드/디스플레이 웨이트를 800·700에서 600으로 일원화**. 이제 전 시스템이 **400(본문)·600(로고·디스플레이·헤드라인·라벨) 2-웨이트**. 로고 E 3선 두께도 14→10.5 동반 하향.
> 로고 상세 → `logo-system.md`.

## 1. 폰트 시스템

| 역할 | 폰트 | 웨이트 | 근거 |
|---|---|---|---|
| 로고 워드마크 | **Paperlogy** | **600** | 각진 기하 골격 + 에디토리얼 개성. E 레이어 커스텀 |
| 디스플레이(헤드라인·간판) | **Paperlogy** | 600 | 큰 조형이 그래픽처럼 |
| 소제목 | **Paperlogy** | 600 | |
| 본문(Body) | **Paperlogy** | **400 · 600** | 재은 지정 — 기본 400, 강조 600 |

- Paperlogy: 무료 상업 사용 가능, 웨이트 100~900(국·영문 한 폰트). CDN: `fonts-archive/Paperlogy`.
- **각진 형태 = “정돈된” 키워드**의 타이포적 표현(컨셉 3키워드 문서 참조).
- 그루비/세리프 이탤릭 액센트는 **사용하지 않음**(톤 통일).
- ※ 장문 가독이 필요한 영역(약관·긴 안내 등)에서 Paperlogy가 무겁게 느껴지면 Pretendard를 **보조 본문**으로 예외 허용 — 단 기본은 Paperlogy.

| 구분 | 자간 |

| 라벨·모드 태그(대문자) | +0.12~0.14em(약간 넓게) |

## 4. 모드 태그 (시그니처)

```
WORK · CLASS · GATHER
```
- Paperlogy 600 / 대문자 / 가운뎃점 구분. 약간 넓은 자간. 로고 3선(레이어)과 개념적으로 연결.

## 5. 운용 규칙

- **가운데 정렬 기본** — 안내문 위주 레이아웃(위/아래 배치, 중앙 정렬). “심플한 레이아웃” 키워드.
- 디스플레이는 크고 과감하게, 글자 조형 자체가 그래픽.
- 강조는 굵기(600)·컬러블록으로. 밑줄·그림자 금지.

## 6. 접근성

- 본문 최소 16px. Paperlogy는 개성이 있어 **초소형 라벨**에선 자간·크기 확보 필수(≥12px, 자간 소폭+).
- 초대형 디스플레이는 대비 확보.
